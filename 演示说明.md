# 10 分钟演示说明

本演示面向“软件体系结构与设计模式”课程期末大作业，目标是展示一个本地可运行、可演示、可扩展的个人发展规划系统原型。

## 一、项目背景，约 1 分钟

介绍项目主题：个人发展规划系统面向大学生四年成长记录场景，帮助学生记录课程、成绩、项目、竞赛、实习、奖项、角色等信息。

可以这样说明：

学生在大学期间会产生大量成长记录，但这些信息通常分散在成绩单、简历、文档和个人记忆中。本系统希望把这些信息统一管理，并进一步转化为能力画像、职业规划和简历材料。

## 二、系统架构，约 1 分钟

说明项目采用分层设计：

- UI 层：`MainWindow` 负责页面展示、交互响应和模块集成。
- 数据模型层：`CourseRecord`、`ExperienceRecord`、`AchievementRecord`、`ProfileInfo` 表示核心业务对象。
- 数据管理层：`DataStore` 负责 JSON 保存读取、GPA 计算、成长建议、能力评分和职业目标分析。
- 可视化组件层：`RadarChartWidget` 使用 `QPainter` 自绘能力雷达图。
- 导出服务层：`ResumeExporter` 负责生成 Markdown 简历。

强调亮点：系统不是把所有代码堆在一个文件里，而是按职责拆分，便于维护和扩展。

## 三、核心功能演示，约 3 分钟

演示顺序建议如下：

1. 打开程序，展示首页总览，包括个人信息、GPA、累计学分、经历数量和成就数量。
2. 进入“课程与 GPA”页面，添加一门课程，展示成绩、学分和绩点记录。
3. 回到首页，说明 GPA 和累计学分会自动更新。
4. 进入“项目/竞赛/实习”页面，添加一条项目或竞赛经历，展示类型、角色、时间和说明。
5. 进入“成就与角色”页面，添加奖项、荣誉或角色记录。
6. 使用“保存数据”和“导入数据”说明系统支持 JSON 数据持久化。
7. 进入“简历导出”页面，展示系统自动生成的简历预览。

演示重点：

- 至少覆盖课程记录、经历记录、GPA 计算、成就记录、数据导入导出、简历导出等核心功能。
- 展示输入校验，例如课程名称不能为空、经历名称不能为空。

## 四、创新功能一：能力雷达图，约 2 分钟

点击“载入演示数据”，回到首页展示能力雷达图。

说明内容：

- 系统会根据已有成长档案自动生成五维能力画像。
- 五个维度包括学业基础、实践能力、竞赛科研、荣誉成果、领导协作。
- 分数不是用户手填，而是由本地规则根据课程、GPA、项目/竞赛/实习、奖项和角色自动计算。
- 雷达图由 Qt 原生 `QPainter` 绘制，不依赖第三方图表库。

可以强调：

能力雷达图回答的是“我现在的发展状态怎么样”。它把原本分散的课程、经历和成就转化成直观的可视化画像。

## 五、创新功能二：职业目标差距分析与四年路线图，约 2 分钟

在首页向下滚动，展示“职业目标差距分析与四年行动路线图”。

演示步骤：

1. 选择“数据分析师”，展示目标匹配度、优势、短板和四年行动路线图。
2. 切换到“保研/科研方向”或“产品/项目管理方向”，说明不同目标会生成不同分析结果。
3. 解释系统如何根据能力分数和目标关键词进行本地规则分析。

说明重点：

- 这个功能不是简单记录数据，而是把系统升级为“规划未来”。
- 它回答的是“我接下来该怎么做”。
- 它不依赖联网或 AI API，保证本地可执行和课堂演示稳定。

## 六、简历导出与总结，约 1 分钟

演示导出 Markdown 简历。

导出的内容包括：

- 个人基本信息
- 课程记录
- 项目/竞赛/实习经历
- 成就与角色
- 能力雷达画像
- 职业目标规划摘要
- 成长画像建议

总结可以这样说：

本系统完成了从成长记录、数据分析、能力可视化到职业规划和简历导出的完整闭环。核心功能满足作业要求，两个创新功能分别体现“当前画像”和“未来规划”，同时系统采用分层结构，便于后续扩展为 SQLite 数据库存储、更多职业目标规则或 Web 版本。

## 七、展示注意事项

- 演示前先确认程序可以正常启动。
- 建议先点击“载入演示数据”，保证雷达图和路线图有完整效果。
- 如需展示输入校验，可以尝试空课程名添加课程。
- 首页内容较多，可以使用滚动条展示下方职业目标路线图。
- 最后导出 Markdown 简历，展示系统输出能力。
